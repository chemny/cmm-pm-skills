# 设计蓝图：市场调研阶段（五看三定）

> 状态：已与 Neo 对齐（2026-06-22），作为改造 `cmm-pm-skills` 市场调研/策略阶段的依据。
> 来源：多轮讨论沉淀（语音输入法案例复盘 → 暴露 Step1 失误 → 引入五看三定）。

---

## 0. 总原则

**一把刀：描述世界 vs 决定我们的应对。**
- 凡"描述别人/现状"的 → 放调研。
- 凡"决定我们自己"的 → 放策略及以后。
- 同一主题（技术/营销/产品设计）会出现两次：调研里=「竞品怎么做的」(观察)，策略里=「我们怎么做」(决策)。不是重复，是观察→决策的两端。

**阶段模型（四段各占一个问题）：**

| 阶段 | 独占问题 | 主干框架 | 判断词 |
|---|---|---|---|
| ① 市场调研 | 外面的世界长啥样？ | **五看** | 这是不是好市场 |
| ② 需求发现 | 用户到底要什么？真不真？ | JTBD + 持续发现 | 这需求值不值得做 |
| ③ 产品策略 | 我们凭什么赢？ | **三定** | 我们怎么打赢 |
| ④ 需求执行 | 做成啥、怎么交付？ | Working Backwards / PR-FAQ | 做什么、做到什么程度 |

---

## 1. Step 0 —— 准入闸门（最关键的改造，上次就死在这）

收不齐关键字段，**不许进搜索**。

**6 必填字段：**
| 维度 | 要问清 |
|---|---|
| 产品形态 | 系统级输入法 / 跨应用听写App / 插件 / ToB SaaS |
| 目标人群**细分** | 哪类人？律师/程序员/销售/高管？场景=写邮件/记会议/写代码？ |
| 市场地域 | 中国 / 海外 / 特定区域 |
| 商业意图 | 免费走量 / 订阅 / ToB 私有化？谁付费？ |
| **参考标杆** | "想做成像 X 那样"——一句话锁定搜索方向（最关键） |
| 约束 | 预算 / 团队 / 时间 / 已有技术或渠道 |

**放行规则：** 前 5 项缺任意一项 → 必须追问，不许用默认值蒙混。用户说"你来定" → 列出我要采用的默认值，让他确认一次再走。

## 2. Step 1 —— 五看（市场调研主体）

每个"看"= 一组定向搜索 + 一个结论。客观描述，不做我方决策。

### 看 ① 行业/趋势
- 市场升还是降？增长率、驱动力、生命周期阶段、技术/政策变量。
- → 回答：趋势 + 容量(TAM/SAM/SOM，**每个数写出计算假设**)。

### 看 ② 市场/客户
- 容量细分、目标人群画像、真实痛点、使用场景、**付费意愿**。

### 看 ③ 竞争（含三个观察子项——上次漏的）
- 直接 / 间接 / 替代竞品 + 份额 + 空白象限（**且要验证空白是真空白**）。
- 子项 a：**竞品技术/产品方案**（云端ASR？本地？端到端大模型？LLM后处理？）
- 子项 b：**竞品营销打法**（获客/定价/口号）
- 子项 c：**竞品产品设计特点**（交互亮点）

### 看 ④ 自己（六问漏掉、五看补回）
- 我方能力、资源、优劣势、**与竞品的差距**。
- 诚实评估：我们凭什么有机会，差在哪。

### 看 ⑤ 机会
- 战略机会点、市场空间、取舍。→ 输出"卡位假设"，接 ② 需求发现。

### 轻量盈利模式扫描（回答"钱"的问题）
- 竞品都怎么收钱、客单价区间、有没有人验证过付费。
- 注意：**只扫描，不做完整商业模式设计**（那是 ③ 的活）。

### 强制：看空 + 自检
- 必须写"**为什么这个机会可能是坑**"一节（进入门槛、合规、技术难度、付费意愿存疑）。
- 交付前**自动过一遍红队自检**（用 strategy-red-team 的思路审自己的报告）。

## 3. 搜索可信度纪律（贯穿 Step1）

- 关键数字必须 **WebFetch 一手源**核对，SEO/博客降级。
- 厂商自夸一律标〔厂商宣称〕。
- 数字冲突要**三角验证**，不许随手挑一个。
- 商业模式/定价要**专门搜**，不能漏。
- 搜索由"五看"维度反向驱动，每个看配一组 query，不是想到啥搜啥。

## 4. Step 2 —— 三定（属于 ③ 产品策略，这里只登记交接）

| 定 | 内容 | 对应你问过的 |
|---|---|---|
| 定战略控制点 | 我们的壁垒/护城河/差异化 | "我们怎么造技术壁垒" |
| 定目标/策略 | 打法、商业模式、定价、路径 | 完整商业模式在这做 |
| 定关键任务 | 落地任务 → 接 ④ 执行 | |
