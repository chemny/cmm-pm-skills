# 需求洞察：sayit（生成型 Agent 语音输入法）

> 产品：**sayit** —— 对着任意输入框说口语 → 直接生成可用书面文本 → 确认插入。人群：泛白领。
> ②职责：验需求真伪 + 给风险结论；**做不做、怎么走，由用户决定。** 全程对齐 sayit，不漂。

---

## 真需求（JTBD）
当我在办公中要写邮件 / 回复 / 消息 / 文案时，我想**说口语就直接得到可用的书面文本**——省去打字，也省去把口语腔反复改成书面的来回。

## 这需求真不真（三角验证，有数据）
| 关 | 结论 | 证据（≥2 类源） |
|---|---|---|
| 痛真 | ✅ | 传统语音输入"转写后口语化、要大量手动改、不能直接用"〔36氪 + 知乎 + CSDN〕——sayit 正解这个 |
| 够频 | ✅ | 语音输入高频：百度日均语音请求 >10 亿次；讯飞语音日活 +18%〔行业/艾媒〕 |
| 趋势对 | ✅ | 大模型正把语音输入从"转写"推向"成稿"：千问过滤语气词、豆包自动润色组织、Wispr 移除冗言〔163 + 掘金 + 36氪〕 |

→ **需求等级：真且高频，方向（成稿/Agent）顺势。**

## ⚠️ 竞争现实（同样有据 —— 风险就在这）
"生成型/成稿型语音输入"**已经是一片在打的市场，头部免费、大厂、中文强**：
- **豆包语音输入法**：GLM 大模型自动润色成稿、说碎片思路→AI 组织、中文最佳、**完全免费**〔掘金/CSDN〕
- **千问语音输入法**：过滤语气词、成稿、宣称效率 +500%〔163〕
- **闪电说**：语音输入 + 技能执行（"一句话执行"）、读屏式上下文回复、**免费、主打办公**〔aihub〕——**它已经在做你说的 Agent 方向**
- Wispr Flow（付费）、Typeless、LazyTyper、Spokenly 等

## 一手验证计划（若继续）
1. **Mom Test 访谈泛白领**：现在用什么语音输入？豆包/千问的成稿够用吗？哪里不满意？（找未被满足的缝）
2. **行为信号**：豆包/闪电说用户吐槽里的未满足点。
3. **小样本对比**：让目标用户对比 sayit 雏形 vs 豆包，看有没有明确偏好理由。
