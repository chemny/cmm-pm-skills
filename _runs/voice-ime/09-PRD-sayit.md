# 产品需求文档（PRD）：sayit V1（PC 端）

| 项 | 内容 |

### 1. 引言与术语
- **sayit**：一个语音输入法。默认"说什么输入什么"；内容较长时自动轻润色；用户显式要求时按其风格个性化改写/生成。
- **原话**：ASR 转写的逐字结果。**轻润色**：去口水词/补标点/顺语序，不改原意、不换说法。**个性化生成**：按用户风格改写或组织成稿。
- **角标**：插入时短暂显示当前处理态（原话/已润色/已生成）。

### 2. 目标与背景
- 痛点（继承②）：办公人士打字慢、语音转写后口语化要手动改〔36氪/知乎〕。
- 差异化（继承③）：豆包/千问做"通用成稿"，sayit 做"**像你写的**成稿 + 数据本地"。
- V1 目标：在 PC 端跑通"语音输入 + 三态处理 + 个性化 + 本地隐私"，验证留存与付费转化。

### 3. 目标用户与场景
- 用户：每天写大量邮件/消息/文案的泛白领（运营、销售、自媒体、管理者等）。
- 主场景：在任意 PC 应用（微信/钉钉/邮箱/Word/浏览器输入框）里，用语音快速、得体地输入文字。

### 4. 成功指标
| 指标 | 目标（对标 Wispr） |
|---|---|
| 北极星 | 周活用户中"用 sayit 完成 >70% 文字输入"的占比 |
| 留存 | 6 周留存 ≥ 行业基准（Wispr 80% 为标杆） |
| 付费转化 | 免费→Pro 转化率（Wispr 19% 为参照） |
| 成稿质量 | 润色/生成后人工二次修改率 ↓ |

### 5. 范围
**V1 做（P0）**：全局唤起录音、ASR 转写、三态处理引擎（原话/轻润色/个性化）、个性化风格引擎+冷启动导入、直接插入+角标+撤销、显式个性化触发、本地隐私存储、freemium 计量、设置。
**V1 不做（P1+）**：移动端、跨设备同步、场景感知（按 App 自动切语气）、团队/ToB 版、皮肤主题。

### 6. 功能清单与产品结构
- **F1 全局唤起与录音**（快捷键，短按开始/再短按停止）
- **F2 语音转写**（ASR）
- **F3 三态处理引擎**（原话 / 轻润色 / 个性化 的判定与执行）—— 核心
- **F4 个性化风格引擎**（学用户风格；onboarding 导入历史文本冷启动）
- **F5 插入与撤销**（直接插入当前输入框 + 角标 + Ctrl+Z 还原原话）
- **F6 显式个性化触发**（语音指令"写成邮件/正式点/按我风格" + 修饰键）
- **F7 本地隐私存储**（风格数据本地加密）
- **F8 Freemium 计量与限制**（免费 2000 字/周；Pro 不限量+个性化深度）
- **F9 设置**（快捷键、阈值、润色开关、账户）

### 7. 用户流程与业务逻辑（核心）
**主流程：**
```
按快捷键(短按) → 开始说话(浮窗显示实时转写)
 → 再按快捷键(短按) → 停止
 → 三态处理引擎判定：
      ① 有"个性化指令/修饰键"? → 个性化生成(风格引擎) → 角标[已生成]
      ② 按住 Shift 短按? → 强制原话 → 角标[原话]
      ③ 转写字数 > 润色阈值(默认 50)? → 轻润色 → 角标[已润色]
      ④ 否则 → 原话 → 角标[原话]
 → 直接插入到当前焦点输入框
 → (可选) Ctrl+Z → 还原为原话
```
**关键业务规则：**
- **默认安全**：无指令、短内容 → 一律原话；轻润色不改原意、不换说法。
- **个性化只在显式触发时发生**（不靠长度自动），避免擅自改写用户的话。
- 阈值（默认 50 字）可在设置调；润色可全局关闭。

### 8. 详细功能需求（P0 + 验收标准）
**F1 全局唤起与录音**
- 行为：全局快捷键（默认可自定义）；短按=开始录音并显示实时转写浮窗；再短按=停止。
- 边界：无麦克风权限→提示授权；录音中再次系统占用→优雅中断并保留已转写。
- AC：任意前台应用下快捷键可唤起；从按下到开始录音 <300ms。

**F2 语音转写**
- 行为：调用 ASR（外采内网/云或本地，见 §13 选型），输出文本。
- 边界：中英混输、专业名词；嘈杂环境标低置信。
- AC：标准普通话字准确率 ≥95%（实测样本）；实时转写句延迟 <1s。

**F3 三态处理引擎**（核心）
- 行为：按 §7 业务规则判定原话/轻润色/个性化并执行。
- 轻润色：去"嗯/那个/就是"等口水词、补标点、顺语序，**保持原意与措辞风格**。
- 边界：含数字/人名/代码/引用时，润色须保留原文 token 不改写。
- AC：①短内容(≤50字)100% 原话直插；②长内容轻润色去口水词且不改原意（实测人工判定改写率=0）；③有指令时走个性化（见 F6）。

**F4 个性化风格引擎**
- 行为：基于用户历史文本/采纳记录，生成"风格画像"（轻量：风格 prompt/记忆，不训大模型），用于个性化生成。
- 冷启动：onboarding 引导导入 3–5 段历史邮件/文章 → 生成初版风格。
- AC：导入样本后，个性化生成结果在盲测中被用户判为"更像我写的" ≥ 对照（无风格）；越用风格越贴（采纳数据回灌）。

**F5 插入与撤销**
- 行为：处理结果直接插入当前焦点输入框；右下角短暂角标显示态；Ctrl+Z 还原原话。
- 边界：目标输入框不支持程序化插入→回退到剪贴板+提示粘贴。
- AC：主流应用(微信/钉钉/邮箱/Word/浏览器)插入成功；Ctrl+Z 一次即还原原话。

**F6 显式个性化触发**
- 行为：识别语音中的指令（"写成邮件""正式一点""按我风格""短一点"等）或修饰键，触发个性化生成。
- AC：说出指令后走个性化生成且角标[已生成]；指令词库可扩展。

**F7 本地隐私存储**
- 行为：风格画像、历史样本、用户语料**本地加密存储(AES-256)**，不上传云端。
- AC：抓包验证个性化数据不外传；卸载无明文残留。（注：ASR/LLM 若走云仅传当次语音/文本，§13 标明，并提供"本地模式"选项的 P1 规划。）

**F8 Freemium 计量与限制**
- 行为：免费版每周 2000 字额度；超出提示升级 Pro；Pro 不限量 + 个性化深度/全部指令。
- AC：字数计量准确；额度用尽有清晰升级引导；Pro 解锁即时生效。

**F9 设置**：快捷键自定义、润色阈值、润色开关、强制原话修饰键、账户/订阅。

### 9. 交互与界面（UX）
- **录音浮窗**：唤起后屏幕一角浮窗，显示实时转写 + 声波动效 + "再按结束"提示。
- **角标**：插入瞬间在浮窗/光标旁显示态（原话/已润色/已生成），~1s 淡出。
- **Onboarding**：首次启动引导设权限、设快捷键、导入历史文本学风格。
- **设置面板**：简洁，快捷键/阈值/订阅。

### 10. 数据需求
- **实体**：用户、风格画像、历史样本、语音会话(临时)、转写文本、处理结果(态/原话/最终)、用量计数、订阅。
- **存储**：风格/样本/用量 → 本地加密 DB；语音 → 处理后即删（不留存）。
- **埋点（指标用）**：唤起次数、各态占比(原话/润色/生成)、采纳/撤销率、二次修改量、字数用量、转化/留存事件。

### 11. 非功能需求
- 性能：唤起 <300ms；实时转写句延迟 <1s；轻润色 <1.5s；个性化生成 <3s。
- 准确率：普通话字准 ≥95%；轻润色"零改写原意"。
- 隐私安全：风格数据本地加密、不外传；语音用后即删。
- 兼容：Windows 10+/macOS 12+；主流应用输入框可插入。
- 稳定：常驻后台低占用，不影响打字。

### 12. 假设、风险、待解问题

### 13. 排期与依赖
- **依赖**：ASR（外采：豆包/讯飞/阿里云 内网或云 API，约 3 元/会议小时级成本〔①〕）；LLM 成稿/个性化（豆包/DeepSeek，token 成本极低〔①〕）。
- **里程碑**：
  - M1（2–3 周）：F1+F2+F3(原话/轻润色)+F5 → "能用的语音输入法"。
  - M2（2–3 周）：F4 个性化+冷启动 + F6 指令 + F7 本地隐私 → 差异化跑通。
  - M3（1–2 周）：F8 freemium + F9 设置 + 埋点 → 可灰度。
