# 技术方案：sayit V1（PC 端）· v2（优化后 ⑥ 重跑）

> 承接 ④PRD + ⑤界面 + ⑦backlog。技术提案，最终架构由工程定；需实测处标〔POC〕。
> 本版重点：新规则(卖点守护/成本延迟/诚实 spike)把 v1 漏的揪了出来——见 §7、§5、文末对比。

## 1. 技术选型（带依据）
| 层 | 选型 | 依据 |
|---|---|---|
| 桌面框架 | **Tauri** | 占用低、托盘/全局快捷键易实现，契合"常驻 CPU<2%/内存<150MB"〔腾讯云/CSDN〕 |
| ASR | **本地优先**(Whisper/FunASR) + 云备选 | 本地语音处理已被验证可行：蛐蛐(本地 FunASR)、superwhisper(本地 Whisper)〔①〕；隐私优先 |
| 润色 LLM | **本地小模型** | 去口水/标点/顺句是简单任务，端侧小模型够用——闪电说即用端侧模型做办公语音〔aihub〕；具体模型/硬件下限〔POC〕 |
| 个性化生成 LLM | **本地小模型默认 / 云增强可选** | 见 §7 冲突解决 |
| 风格库 | 本地加密 SQLite | 隐私(F7) |

```mermaid
  ROUTE -->|"短/Shift"| RAW["原话 (无需LLM)"]

  INJ --> UNDO["Ctrl+Z 还原原话"]

```

## 3. 数据模型与存储
- 风格画像(结构化：用词/句长/语气/称呼/禁忌 + few-shot 样本)、历史样本、用量、订阅；**本地 AES-256**；语音处理后即删。

## 4. 关键技术决策
- **本地 vs 云(核心)→ 分层混合**：见 §7（这是全方案最重要的决定）。
- Tauri vs Electron → Tauri（占用硬指标）。
- 个性化 → 结构化画像 + few-shot + RAG，**不做 per-user 微调**（成本/隐私/冷启动更优）。

## 5. 难点 / SPIKE 解决（诚实：哪些真解了、哪些只是方向）
- **SPIKE-P 个性化**：方向 = 结构化风格画像 + few-shot + RAG；**但"画像 schema + 怎么从历史文本抽取"仍是设计子 spike**（起步：用 LLM 按定义 schema 抽取；schema/抽取 prompt 待设计）。效果〔POC〕= S7d 盲测≥65%。→ **部分解，剩余子 spike 已标清，不假装全解。**
- **指令检测(三态"有指令"分支)**：v1 当已解，**其实是 NLU 子 spike** → 起步用**固定指令集/唤醒短语**("写成邮件/正式点/短一点"关键短语匹配)，自然语言任意指令留 P1。
- **本地小模型选型**（哪个模型、硬件下限、CPU-only 可行性）〔POC〕。
- **全 App 文本注入**：Accessibility/输入法级注入 + 剪贴板兜底，逐 App〔POC〕。

## 6. 接口 / API
- ASR：`audio → {text,timestamps,confidence}`（本地/云同契约，可换源）。
- 处理：`{mode:polish|generate, text, style_profile?, instruction?} → {output}`（本地/云同契约）。

## 7. 非功能 + 冲突检查（本版核心，新规则产出）
**⚠️ 架构 vs 卖点冲突（必查）：**
- **矛盾**：sayit 卖点 = 本地隐私 + 个性化；但个性化生成要 LLM。**纯云 LLM → 你的原话 + 风格画像上云 → 打脸"数据归你"**（v1 正是这个漏洞）。
- **解（分层混合）**：
  - **原话/轻润色（高频）= 本地小模型**：即时、零边际成本、数据不出域；
  - **个性化生成（低频、用户显式触发）= 本地小模型默认（隐私优先），可选"云增强"**（更高质量，明确告知 + 用户授权才上云）；
  - **风格画像始终本地**。
  - → 隐私卖点守住、高频路径快且免费、云只在 Pro 自愿增强。**卖点与架构不再打架。**

**延迟预算：** 原话 <100ms(本地)；润色 目标<800ms(本地小模型；CPU-only〔POC〕)；生成 1–3s(低频、用户在等成稿，可接受)。

**系统级成本（对齐 freemium）：** 高频润色走本地 → **边际成本≈0** → **免费版 2000字/周可持续**；云生成仅 Pro 自愿用时产生 → 成本随付费走。
> 对比 v1（全云 LLM）：每条中长输入都触发付费云调用 → 免费版会亏。**新规则当场把这个成本坑算了出来。**

## 8. 风险 / 依赖 / 未决
| 项 | 解法/状态 |
|---|---|
| 本地小模型质量/硬件门槛 | 〔POC〕选型 + CPU-only 兜底;不达标则该路径降级云(用户授权) |
| 个性化"更像你"效果 | 〔POC〕S7d 盲测,不达标加 RAG/调 schema |
| 全 App 注入兼容 | 〔POC〕+ 剪贴板兜底 |
| 指令检测 | 起步固定指令集,自然语言留 P1 |

## 9. 实施要点
- M1：Tauri 壳 + 快捷键 + 录音 + 本地 ASR + 原话/本地润色 + 注入（**全本地、零云成本的可用闭环**）。
- M2：本地个性化生成 + 风格画像（先过 S7d 盲测）+ 可选云增强。
- 先 POC：本地小模型可行性、全 App 注入、个性化效果。

---
## 本次重跑 vs v1（验证 ⑥ 优化是否有效）
1. **v1 漏的"隐私 vs 云矛盾"** → v2 被"卖点守护"揪出，用**分层混合**正面解（高频本地、生成本地默认云可选、画像本地）。
2. **v1 没算的延迟/成本** → v2 给了预算，并发现"**本地润色让 freemium 成本成立**"这个关键结论。
3. **v1 当已解的个性化/路由** → v2 诚实标出**两个隐藏子 spike**(画像抽取、指令检测)。
→ **优化有效：新规则确实把上一版的坑自动逼了出来。**
