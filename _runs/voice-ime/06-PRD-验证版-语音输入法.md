# 产品需求文档：私有化会议纪要 MVP（验证版）

> **验证版**：范围窄（单垂直·律所·核心流程），目标是验证"监管行业愿付费"；但**所建部分功能/流程/验收全 spec 到可开发**。需求验证通过后扩为完整产品。
> 日期：2026-06-22 ｜ 输入：①②③ ｜ 平台：Windows/macOS 桌面端 + 单机/内网部署

---

## 1. 摘要
给律所做一个**数据不出内网**的桌面端会议纪要工具：本地录音/导入 → 本地转写 → AI 生成结构化纪要 → 校对 → 导出。用一个能跑的真实最小产品，验证律所愿不愿付费。

## 2. 背景（继承①②③）
会议纪要 82% 职场人痛、2h 会议手工整理 3-4h〔②〕；监管行业要私有化（飞书要私有化才赢大客户、金融信创 2000亿）〔①③〕；已有智在记录/必捷占位，须在律所垂直做深〔③〕。

## 3. 范围
**做（P0 核心流程）**：本地录音/导入、本地转写+说话人分离、AI 结构化成稿、法律术语库、编辑校对、本地加密存储、导出、单机/内网部署。
**不做（非目标）**：移动端、多租户 SaaS、实时多人协作、等保/信创全认证、跨会议平台插件、自研 ASR。

## 4. 功能清单
- F1 音频输入：本地录音（系统声+麦克风）/ 导入音频文件
- F2 本地语音转写：ASR（内网/私有部署）→ 带时间戳文本 + 说话人分离
- F3 AI 结构化成稿：LLM → 摘要 / 讨论要点 / 待办（责任人+时间）/ 决议
- F4 法律术语库：导入律所术语/案号/当事人，提升识别与成稿
- F5 编辑校对：纪要可改，与转写原文对照、点击定位
- F6 本地加密存储：会议/音频/纪要存本地库（AES-256）
- F7 导出：docx / pdf（套律所纪要模板）
- F8 私有化部署：单机版或内网服务器，数据全程不出内网

## 5. 用户流程（端到端）
```
内网打开应用 → 本地登录
  → 新建会议（标题/参会人/选"法律"术语库）
  → 开始录音  或  上传已有音频
  → 会议中：本地实时转写滚动显示（会后导入则批量转写）
  → 结束 → 自动生成结构化纪要（摘要/要点/待办/决议）
  → 校对编辑（左转写原文｜右纪要，点句定位）
  → 导出 docx/pdf  或  存入本地库
全程：音频与文本不出内网
```

## 6. 详细功能需求（P0 + 验收标准）
**F1 音频输入**
- 行为：录制系统声+麦克风混音，16kHz+；支持导入 mp3/wav/m4a。
- 边界：无麦克风权限→提示并仅录系统声；超 4 小时→自动分段。
- AC：能录满 2 小时不丢帧；导入 1 小时音频成功进入转写队列。

**F2 本地语音转写 + 说话人分离**
- 行为：调内网/私有 ASR，输出带时间戳与发言人标签的文本。
- 边界：多人抢话→尽力分离，不确定标"发言人?"；方言→标低置信。
- AC：标准普通话字准确率 ≥95%（实测样本）；说话人分离对 3-5 人会议可用；**全程无外网请求（抓包验证）**。

**F3 AI 结构化成稿**
- 行为：把转写文本经 LLM 生成四块——摘要、讨论要点、待办（含责任人/时间）、决议。
- 边界：信息不足的块输出"无"；不臆造待办/责任人（CONVENTIONS 同源纪律）。
- AC：一份 1 小时会议在 ≤2 分钟出结构化纪要；待办项能正确抽出责任人与时间 ≥80%（实测）。

**F4 法律术语库**
- 行为：导入 csv 术语/案号/人名，转写与成稿优先匹配。
- AC：导入术语后，该批术语识别错误率较通用明显下降（A/B 实测）。

**F5 编辑校对**
- AC：纪要任一句可编辑；点纪要句→高亮对应转写原文位置；改动实时本地保存。

**F6 本地加密存储**
- AC：音频/转写/纪要落本地加密库（AES-256）；卸载/拷库无明文泄露。

**F7 导出**
- AC：导出 docx/pdf 套用律所模板，格式不乱。

**F8 私有化部署**
- AC：单机版可**离线**完整跑通 F1–F7；或内网服务器版数据不出域（必捷已证单机私有化可行〔③〕）。

## 7. 数据与部署
- **实体**：会议(id/标题/时间/参会人/术语库)、音频(本地路径/时长)、转写(分段/时间戳/发言人)、纪要(摘要/要点/待办/决议/版本)、术语库。
- **存储**：本地加密 DB（SQLite+加密）+ 本地文件；不上云。
- **部署**：① 单机一体机（律所一台机离线用）；② 内网服务器版（多人内网访问）。ASR/LLM 走**内网私有部署或内网 API**，不出域。

## 8. 非功能需求
- 准确率：普通话字准 ≥95%；法律术语错误率 < 通用工具。
- 延迟：实时转写句延迟 <3s；1h 会议成稿 ≤2min。
- 安全：数据不出内网、本地加密、操作可审计。
- 可用：单机版断网可完整运行。

## 9. 待解问题 + 排期
| 待解 | owner |
|---|---|
| ASR 选型（外采内网部署 vs 开源 WhisperLiveKit〔③〕） | ④ 技术 |
| 律所纪要模板字段（找 1 家律所要真实模板） | ②+律所 |
| 试点价位 | ②一手验 |

**排期**：第 1-2 周 F1-F3 跑通（录→转→成稿）；3-4 周 F4-F7 + 单机部署；5-6 周给 1-3 家律所**付费试点**，收 ④ 成功阈值数据。
**出口**：试点达标 → 扩完整产品 PRD（多垂直/认证/协作）；不达 → PIVOT/KILL。
