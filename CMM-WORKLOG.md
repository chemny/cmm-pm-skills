# write-prd 改造笔记

> 每条改造都在这里记"为什么改"，方便日后回溯，也是你这套方法论的沉淀。

---

## 一、原版是什么(v0 基线)

- 来源：`pm-skills/pm-execution/skills/create-prd/SKILL.md`(86 行)
- 形态：一个 **skill**(知识/模板)，由 `commands/write-prd.md` 这个命令调用
- 核心：一套 **8 段式 PRD 模板**(摘要/联系人/背景/目标/市场细分/价值主张/方案/发布)
- 风格：英文、要求"写给小学毕业生看"、用 OKR+SMART、Value Curve 框架

---

## 二、通用性问题(我的初判 — 待你确认/补充)

| # | 问题 | 我的看法 | 你的决定 |
|---|---|---|---|
| 1 | 全英文 | 你的场景要中文输出，至少双语 | ☐ |
| 2 | 8 段模板偏西方咨询味(Value Curve / SMART OKR) | 不一定贴你的研发习惯，可换成你惯用的结构 | ☐ |
| 3 | "写给小学毕业生" | 你的读者是研发团队，语域可能要更专业/精确 | ☐ |
| 4 | 缺"需求来源 / 优先级"段 | 真实 PRD 要说清需求从哪来、为什么排这个优先级 | ☐ |
| 5 | 缺"验收标准(AC)" | 这是交给研发/测试的关键接口，原版没有 | ☐ |
| 6 | 缺"数据埋点 / 指标定义" | 闭环第6步"数据"要靠这里预先定义，原版只在目标里提 KR | ☐ |
| 7 | 缺"风险 / 依赖 / 排期接口" | 跟研发交接(图里第2→3步)需要 | ☐ |
| 8 | 与 `/discover` 的衔接 | 原版假设你已想清楚，没接 discovery 产出 | ☐ |
